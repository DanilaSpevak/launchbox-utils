# Roadmap

## Архитектура / сопровождаемость

- [ ] Разделить `launchbox_tools/gui/app.py` на более мелкие компоненты: общий каркас окна, панели операций, контроллер запуска операций и обработчики состояния.
- [ ] Вынести общий GUI-код запуска фоновых задач, логирования, обработки ошибок и блокировки повторного запуска в отдельный слой.
- [ ] Описать в `ARCHITECTURE.md` целевую структуру GUI после декомпозиции, чтобы новые операции не раздували один файл.

## Безопасность / надежность

- [ ] Перевести безопасную запись XML на уникальные временные файлы рядом с целевым XML и гарантированную очистку временных файлов при ошибке.
- [ ] Усилить подтверждение apply-операций: показывать список затрагиваемых XML-файлов, ожидаемое число изменений и путь к резервной копии.
- [ ] Проверить, что все будущие XML-модифицирующие операции проходят через dry-run, `ensure_safe_to_mutate()` и `safe_write.py`.

## Тесты / quality gates

- [ ] Разделить большой `test/test_launchbox_utils.py` по подсистемам: config, CLI, audit, dedupe, path replacement, reports, GUI.
- [ ] Добавить `ruff` как базовую проверку форматирования и типичных ошибок.
- [ ] Добавить `mypy` в постепенном режиме для наиболее стабильных модулей: models, paths, config, operations.
- [ ] Закрепить smoke-тест GUI layout и фонового логирования как отдельный быстрый тест.

## UI / UX

- [x] Явно маркировать или скрывать запланированные, но еще не реализованные операции, чтобы они не выглядели сломанными.
- [ ] После каждой операции показывать компактную сводку: обработано платформ, найдено проблем, изменено файлов, куда записаны отчеты.
- [ ] Добавить прогресс и возможность отмены для долгих операций там, где это можно сделать без риска повредить данные.
- [ ] Унифицировать кнопки выбора и открытия папок: заменить технические `...` / `>` на понятные обозначения и одинаковые подсказки.

## Документация / кодировка

- [ ] Проверить и исправить русские тексты в `README.md` и `ROADMAP.md`, чтобы файлы корректно читались как UTF-8.
- [ ] Зафиксировать в документации, что roadmap и пользовательские тексты хранятся в UTF-8.
- [ ] Синхронизировать описание GUI в README с текущими группами операций и планируемыми изменениями.

## Будущие функции

- [ ] Добавить восстановление основного файла из `AdditionalApplication` при его отсутствии.
- [ ] Добавить выгрузку избранного в структуру папок для портативки с выбором режима: только основной файл или все файлы.
- [ ] Реализовать поддержку многодисковых игр.
